# TH3V151T0R5_B 전투봇

마스토돈 기반 크리쳐 레이드 자동 정산 봇.
`[전투시작]` 툿/DM → 러너 행동 수집(DM) → 전투 계산 → 결과 송출 → 시트 자동 갱신.

---

## 구조

```
main.rb               진입점. 타임라인/DM 폴링, 라운드 진행, 정산
sheet_manager.rb      구글 시트 읽기/쓰기 (스탯, 현황, 맵현황, 보스)
battle_calculator.rb  명중/회피/크리티컬/피해/좌표 계산
mastodon_listener.rb  퍼블릭 툿 및 DM 전송
```

---

## 전투 흐름

1. `[전투시작] @러너1 @러너2 ...` 툿 → 전투 시작 (태그된 계정이 참여자, `[숫자]` 포함 시 시작 라운드)
2. 봇이 안내 송출 → 러너는 **DM**으로 행동 제출 (1인 1라운드 1회)
3. 라운드 종료: **전원 제출** 또는 **5분 경과** (미제출자 "턴 상실")
4. 정산 → 결과 송출 → **60초 후 다음 라운드 자동 시작**
5. 종결: 크리쳐 격파(승리) / 러너 전원 전투불능(패배) / `[전투종료]`(강제 종료)

### DM 테스트 모드

봇에게 **DM**으로 `[전투시작] @러너1 @러너2` 를 보내면 안내·결과까지 전부 단체 DM으로 진행됩니다.
(태그 없으면 보낸 본인이 러너. DM `[전투종료]`로 강제 종료)
공개 툿으로 시작하면 기존처럼 공개 모드로 진행됩니다.

---

## 행동 커맨드 (DM)

| 커맨드 | 효과 |
|---|---|
| `[공격/크리쳐]` | 크리쳐 공격 판정 |
| `[회복/아이디]` | 대상 회복 (회복량 = 시전자 마법능력, 최대체력 초과 불가) |
| `[방어/아이디]` | 이번 라운드 대상이 받는 피해 절반 |
| `[이동/좌표]` | **가로/세로/대각선 1칸**만 이동 (A1~G8, 즉시 맵 반영) |

이동 별칭: `스토디시`=E7, `A`~`G`=A1~G1 (별칭도 1칸 제한 적용)

---

## 정산 순서

```
기숙사 패시브 → 회복 → 방어 → 러너 공격 → 크리쳐 반격
```

## 계산식

| 판정 | 식 | 비고 |
|---|---|---|
| 명중률 | 60 + 기술 (%) | 실패 시 빗나감 |
| 회피율 | 대상 민첩 × 2 (%) | 명중 후 자동 판정 |
| 크리티컬 | 행운 × 2 (%) | 성공 시 마법능력 2배 |
| 피해 | 마법능력 − 대상 내구도 | 최소 0, 방어 대상 ÷2 |

크리쳐 체력은 전투 시작 시 스탯에서 1회 로드 후 라운드 간 메모리 유지 (시트 원본 보존).
크리쳐 반격은 매 라운드 생존 러너 1명 랜덤.

---

## 기숙사 패시브

스탯 탭 L열(기숙사) + M열(패시브선택 1/2)로 지정.

| 기숙사 | 1번 | 2번 |
|---|---|---|
| 그리핀도르 | 공격자가 정면(방향 기준)에 있으면 내구도 1.5배 | 건강 50% 미만 시 마법능력 +50% |
| 슬리데린 | 이전 라운드 무피해 시 마법능력 +50% (2라운드부터) | 행동 미제출 라운드마다 다음 라운드부터 행운 +10 (누적) |
| 래번클로 | 적에게 상태이상이 있으면 마법능력 +50% | 이전 라운드와 행동 분류가 다르면 기술 +10 |
| 후플푸프 | 이전 라운드 피격 시 내구도 +50% | 전투 중 1회 건강 0 이하 방지 (1로 생존) |

패시브 발동은 결과 툿의 `[기숙사 패시브]` 항목에 표기됩니다.

## 시트 구조

### 스탯/보스 시트 (CREATURE_SHEET_ID)

| 탭 | 열 |
|---|---|
| 스탯 | B=이름 C=건강 D=내구도 E=마법능력 F=민첩 G=기술 H=행운 I=스킬1 J=스킬2 K=방향(상/하/좌/우) L=기숙사 M=패시브선택 (2행부터, 크리쳐 포함) |
| 보스 | 활성화할 크리쳐 행에 TRUE 체크 → 인접 셀의 이름을 스탯 탭에서 조회 |

### 현황 시트 (VIEW_SHEET_ID)

| 탭 | 범위 | 내용 |
|---|---|---|
| 현황 | E4:I11 | 이름 / 위치 / 체력바 / 최대체력 / 상태 (봇 자동 갱신) |
| 맵현황 | B3:H10 | 7×8 그리드에 러너·크리쳐 위치 (봇 자동 갱신) |

## 설치 및 실행

```bash
git clone https://github.com/CriminalTalent/TH3V151T0R5_B.git
cd TH3V151T0R5_B
bundle install --path vendor/bundle
cp /path/to/.env .env
cp /path/to/credentials.json credentials.json
pm2 start main.rb --name battle_bot --interpreter ruby
pm2 save
```

봇 재시작 시 기존 타임라인·DM은 스냅샷 처리되어 과거 `[전투시작]` 툿에 재반응하지 않습니다.
