# 전투봇 핵심 룰 가이드 (변경 금지 기준)

이 문서는 코드로 직접 검증된 전투 메커니즘을 정리한 것입니다. 향후 수정 시 이 문서와 어긋나면 안 되며, 룰을 바꾸려면 이 문서를 먼저 갱신해야 합니다.

---

## 1. 체력 / 최대체력

- 체력바는 **현재 체력** 기준으로 채워진다 (최대체력이 아님).
- 스탯 시트의 **K열("최대건강")이 최대체력의 유일한 출처**다. E열("건강")은 현재체력이며, 최대체력에 영향을 주지 않는다.
- 활력/회복/구원 등 회복 스킬은 **최대체력을 절대 초과하지 않는다.**

## 2. 표시이름 / ID

- 전투 로그(공격/이동/회피/보호막/전투불가 등)에는 계정 ID가 아니라 **캐릭터 표시이름**이 출력되어야 한다.
- 이는 스탯 시트 B열("이름")에서 온다. 표시이름이 비어있으면 ID로 폴백한다.

## 3. 희생 (커버)

- 희생은 대상을 지정해 자신이 대신 맞도록 등록하는 스킬이다.
- **공격자는 명중 판정을 해야 하며, 명중해도 커버한 캐릭터가 회피할 수 있다.**
- 커버는 보스의 모든 공격 유형(기본 반격, 전체공격, 범위공격, 지정공격 등)에 동일하게 적용되어야 한다.

## 4. 보호 / 방어

- 보호(보호막 부여), 방어(내구도 배수)는 **명중 판정 없이 즉시 적용**된다.
- 보호막은 **라운드가 지나도 잔여분이 유지**되며, 재시전 시 **누적**된다 (덮어쓰지 않음).

## 5. 강화

- 강화는 대상의 마법능력을 올리는 버프다.
- **중첩되지 않는다** — 재시전 시 이전 값을 대체할 뿐, 더해지지 않는다.

## 6. 고육지책

- 자신의 건강을 소모해 그만큼(1:1) 마법능력을 올려 공격한다. 예: 60 소모 → 마법능력 +60.
- 명중률 100% (반드시 명중), 전투 중 1회 제한.

## 7. 판정 확률 공식

- 명중률 = 60 + 기술 (%)
- 회피율 = 대상 민첩 × 1% (또는 스킬별 명시값)
- 크리티컬률 = 행운 × 1% (또는 스킬별 명시값)
- 모든 확률은 0~100% 범위로 제한된다.

## 8. 방향 / 머리 위치

- 크리쳐는 상/하/좌/우 중 하나의 방향(facing)을 가진다.
- 다중 셀(예: 3x1) 크리쳐라도 **`creature[:pos]`(앵커 좌표)가 항상 머리이며, 전방 판정의 기준점**이다.
- 준비 라운드 안내에는 점유칸 목록에서 머리 칸을 명시적으로 표시해야 한다.

## 9. 세션 / 타래

- 보스행동커맨드는 **타래(in_reply_to_id) 기준으로 세션을 특정**해야 한다. 타래가 불명확하고 활성 세션이 여럿이면 적용을 보류한다.
- 좌표 지정/전체공격형 명령은 **활성 세션이 유일할 때만** 적용 가능하다 (오배정 방지).
- 동일 툿(status id)은 어떤 수신 경로(공개/멘션/DM)로 들어와도 **한 번만 처리**한다.
- 분할 게시된 안내 툿은 **모든 조각의 ID가 세션에 등록**되어야 하며, 어느 조각에 답해도 같은 세션으로 인식되어야 한다.
- 안내 게시가 **부분적으로만 성공하면 완전 성공으로 간주하지 않는다** — 재시도한다.

## 10. 429 대응

- Mastodon API가 429를 반환하면 `Retry-After` 헤더를 확인해 대기 후 재시도한다 (최대 3회).
- 재시도 소진 시 실패로 기록하고 무한 반복하지 않는다.

---

## 수정 시 지켜야 할 원칙

1. **코드를 먼저 읽고 검증한 뒤에만 "고쳤다"고 표시한다.** 추측이나 재확인 없는 "완료" 표기 금지.
2. 스탯/시트 컬럼 구조를 바꾸는 수정은 반드시 실제 시트 헤더를 재확인한 뒤 진행한다.
3. 로그 문구를 바꿀 때는 ID/표시이름 어느 쪽을 쓰는지 명확히 구분한다 (내부 키 조회는 ID, 사용자 노출 텍스트는 표시이름).
4. 문법 검증(`ruby -c`) 없이 서버에 반영하지 않는다.
5. 이 문서(GAME_RULES.md)와 STANDARD.md는 실제 수정이 있을 때마다 함께 갱신한다.

## 이번 세션 전체 수정 내역

### 긴급 버그
1. 준비 라운드/보스 턴 안내문 문법 오류로 봇이 죽던 문제 → `main.rb` 수정
2. 최대건강(K열) 미반영 — 회복/구원이 항상 "+0"으로 나오던 근본 원인 → `sheet_manager.rb`
3. 표시이름(display_name) 필드가 runner_state에 아예 없어 모든 로그가 ID로 나오던 근본 원인 → `sheet_manager.rb`, `battle_state.rb`
4. 희생(커버)이 보스의 실제 패턴 공격(전체공격/범위공격/지정공격 등)에서 전혀 작동하지 않던 문제 → `battle_boss_patterns.rb`

### 밸런스/로직 버그
5. 고육지책 마법능력 보너스 공식 오류 (0.5배로 계산되던 것 → 1:1로 수정) → `battle_round.rb`
6. 강화 재시전 시 중첩되던 버그 (대체되어야 함) → `battle_round.rb`

### 표시/UX 개선
7. 데미지/이동/반격 로그 전체를 ID → 표시이름으로 변경 → `battle_round.rb`, `battle_boss_patterns.rb`
8. 정산 결과에 크리쳐 방향 표시 추가 → `battle_round.rb`
9. 준비 라운드에 다중 셀 크리쳐의 머리 위치(점유칸 중 어느 칸인지) 표시 추가 → `battle_grid.rb`
10. 다중 세션 진행 시 준비 라운드 안내에 레이드 전체 몹 현황 표시 추가 → `main.rb`

### 안정성 개선
11. 429 Too Many Requests 대응 — Retry-After 확인 후 최대 3회 재시도 (POST/GET 모두) → `mastodon_client.rb`
12. 분할 안내 툿의 모든 조각 ID를 세션에 등록 (기존엔 마지막 조각 ID만 등록) → `main.rb`
13. 안내 게시 부분 성공 시 완전 성공으로 오판하지 않고 재시도하도록 수정 → `battle_api.rb`, `main.rb`
14. 로그 세션 ID 누락 1곳 수정 (정산 실패 안내 송출 실패) → `main.rb`
