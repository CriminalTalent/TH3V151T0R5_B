## 이번 세션에서 발견하고 수정한 버그

1. **준비 라운드/보스 턴 안내문 문법 오류** — `점유칸: ...join(` 뒤가 끊겨 있어 봇이 죽던 문제
   → `main.rb` 141, 381줄 수정

2. **고육지책 마법능력 보너스 공식 오류** — `(소모량/10)×5`로 계산되어 문서 기준(1:1)의 절반만 적용되던 문제
   → `battle_round.rb`에서 `bonus = cost`로 수정

3. **희생(커버) 미적용** — 보스의 실제 패턴 공격(전체공격/범위공격/지정공격 등)에서 희생이 전혀 작동하지 않던 문제. 기본 반격(패턴 미지정 시)에만 구현되어 있었음
   → `battle_boss_patterns.rb`에 커버 리다이렉트 + 명중/회피 판정 추가

4. **강화 중첩 버그** — 강화를 재시전하면 이전 버프가 새 버프에 더해져 누적되던 문제 (중첩 스킬 아님)
   → `battle_round.rb` `:atk_buff_area` 로직에서 기존 보너스를 차감 후 재적용하도록 수정

5. **정산 결과에 크리쳐 방향(머리 방향) 미표시**
   → `battle_round.rb` 정산 결과문에 방향 추가

6. **최대건강 컬럼 미반영** — 스탯 시트 K열("최대건강")이 있는데도 안 읽고, 현재건강(E열) 값을 최대건강에도 그대로 복사해서 쓰던 문제. 회복/구원이 항상 "+0"으로 나오던 근본 원인
   → `sheet_manager.rb` `read_base_stats`에서 K열(row[10]) 실제로 읽도록 수정

7. **표시이름(display_name) 자체가 runner_state에 없던 문제** — `display_name_of` 헬퍼는 이미 있었지만, `read_runner_state`/`build_fallback_runner_state`가 애초에 `display_name` 필드를 담지 않아 항상 ID로 폴백되던 근본 버그
   → `sheet_manager.rb`, `battle_state.rb`에 `display_name` 필드 추가

8. **데미지/이동/반격 로그가 ID로 출력** — 위 7번 수정과 함께, 공격 헤더/고육지책/습격 이동/행운부여 이동/반격(회피·보호막·필사즉생·후플푸프·전투불가)/보스→러너 피해(회피·보호막·전투불가·희생) 로그 전부 표시이름 사용하도록 수정
   → `battle_round.rb`, `battle_boss_patterns.rb`

9. **준비 라운드 위치 안내 개선**
   - 다중 세션(동시 레이드) 진행 시, 준비 라운드 안내에 전체 활성 몹의 위치/크기/방향을 한 번에 표시 (`레이드 전체 몹 현황`)
   - 다중 셀(예: 3x1) 크리쳐의 점유칸 중 정확히 어느 칸이 머리(전방 판정 기준점)인지 표시 (`C2(머리 ▼), D2, E2` 형식)
   → `main.rb`, `battle_grid.rb`
